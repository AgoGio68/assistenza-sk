Manuale Tecnico: Rilascio e Attivazione v1.6.0

Premessa

Questo documento descrive la procedura **identica e ripetibile** per trasferire l'applicazione su un nuovo server Firebase (Cliente) e attivare la licenza software. Il sistema utilizza un algoritmo di offuscamento per proteggere il codice sorgente da tentativi di copia non autorizzata.

Requisiti minimi sul PC di chi esegue i comandi:
  • Node.js installato (versione 18 o superiore).
  • Firebase CLI installato (`npm install -g firebase-tools`).
  • Accesso amministrativo alla cartella del progetto.

FASE 1: Preparazione e Pubblicazione AZIONI SVILUPPATORE

1 Configurazione dell'Ambiente per il nuovo Cliente

Assicurati di avere i parametri del nuovo progetto Firebase del cliente (es: progetto-pippo-123).

File da modificare: .env
VITE_FIREBASE_API_KEY=AIzaSyA...
VITE_FIREBASE_AUTH_DOMAIN=progetto-pippo-123.firebaseapp.com
VITE_FIREBASE_PROJECT_ID=progetto-pippo-123
VITE_FIREBASE_STORAGE_BUCKET=progetto-pippo-123.appspot.com
VITE_FIREBASE_MESSAGING_SENDER_ID=1234567890
VITE_FIREBASE_APP_ID=1:1234567890:web:abc123def456

2 Compilazione Blindata (Offuscamento)

Esegui la compilazione del codice. In questa fase il sistema trasformerà il codice leggibile in codice protetto per evitare furti di proprietà intellettuale.

npm run build

3 Collegamento al progetto del Cliente

Se non lo hai già fatto, aggiungi il nuovo progetto alla riga di comando di Firebase:

firebase use --add

Seleziona il progetto del cliente dalla lista e dai un nome breve (es: pippo).

4 Pubblicazione (Deploy)

Carica il sito e le funzioni cloud sui server del cliente:

firebase deploy

A questo punto, aprendo il sito del cliente, l'app funzionerà ma mostrerà la schermata di blocco "Sblocco Sistema Richiesto".

FASE 2: Inizializzazione Database AZIONI CLIENTE / SETUP

Questa fase serve a creare in automatico l'utente SuperAdmin e le impostazioni nel database del cliente.

5 Estrazione Chiave Account di Servizio

  • Accedi alla Console Firebase del cliente.
  • Fai clic sull'icona dell'ingranaggio (Impostazioni progetto).
  • Seleziona la scheda Account di servizio.
  • Fai clic sul pulsante blu "Genera nuova chiave privata" in basso.
  • Verrà scaricato un file .json (es: progetto-pippo-firebase-adminsdk-xxxxx.json).
  • Rinomina questo file esattamente in serviceAccountKey.json.

6 Esecuzione Script di Autoinstallazione

Posiziona i file setup-client.cjs e serviceAccountKey.json in una cartella, apri il terminale della cartella ed esegui:

node setup-client.cjs

Cosa succede: Lo script scriverà nel database l'utente admin@admin.it con password admin e imposterà il ruolo SuperAdmin.

FASE 3: Scambio Codici e Attivazione Licenza

7 Generazione Richiesta (Lato Cliente)

Il cliente apre l'indirizzo dell'app (es: https://pippo.web.app).

Visualizzerà nel riquadro grigio il "CODICE RICHIESTA" (es: A1B2-C3D4-E5F6-G7H8).

Il cliente ti comunica questo codice.

IMPORTANTE: Ogni Codice Richiesta è legato strettamente all'ID Progetto Firebase. Se si cambia progetto, serve un nuovo sblocco.

8 Generazione Sblocco (Lato Tuo)

Sul TUO computer, esegui il generatore privato:

node keygen.cjs

Incolla il codice inviato dal cliente. Il terminale ti restituirà il "CODICE DI SBLOCCO".

9 Attivazione Finale

Invia il Codice di Sblocco al cliente. Il cliente lo inserisce nel campo "Codice di Sblocco" dell'app e preme "Sblocca Sistema".

RISULTATO: L'app si sblocca istantaneamente. Il cliente può ora accedere con admin@admin.it e password admin per iniziare a lavorare.
